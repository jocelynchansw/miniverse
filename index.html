Field Manual · Appendices

How this universe was built

One project manager. One AI pair. A small fleet of scoped AI agents. No hand-written code — and no unreviewed code either. The Miniverse is a working demonstration of a person product-managing software into existence: setting direction, writing briefs, drawing boundaries, reviewing everything, and sending work back when the tongue looked like a pancake.

The team

  • The Director (human) — product vision, references, taste, and quality control. Approved every visual against screenshots. Issued the pivotal correction of the project: "the tongue is so out of place."
  • The Pair (Claude) — orchestration and integration: turned direction into briefs, spun up agents, verified their work end-to-end in a live browser, and did the fine sculpting between drafts.
  • Design agent — synthesized three reference sites into a full design spec: palette, typography, microcopy, and the "field manual" conceit you are reading right now.
  • Architecture agent — read the whole codebase and produced the build plan: physics approach, module boundaries, and eight milestones that each left the site runnable.
  • Two builder agents — one owned behavior (world, physics, driving), one owned presentation (this UI). Each got an explicit do-not-touch file list and a contract of interfaces that had to survive. Deviations had to be reported, not hidden.

The process, in eight releases

  1. Prototype on day one. A design-board web page with palette, type, and a fake orbit demo — reviewed and approved before any 3D code existed.
  2. Ship the smallest lovable thing. A fluffy sculpture in space with three orbiting facts. Cute, and finished in one sitting.
  3. Add verbs. User feedback: "it's quite useless." Diagnosis: nothing to do. Fetch and feeding were built as the two highest-delight, zero-backend interactions.
  4. Pivot on reference. New inspiration (a drivable-car portfolio site) reframed the product: stop looking at the dog; become the dog.
  5. Parallelize the thinking. Design spec and technical plan were produced by two agents simultaneously — independent perspectives, one afternoon.
  6. Serialize the building. Behavior first, presentation second, with file-boundary contracts between them. One agent stalled mid-build; it was resumed from its last reported state rather than restarted, losing nothing.
  7. Critique like it ships tomorrow. A structured design review produced six prioritized fixes (mobile camera framing was the critical one). All six shipped in the next rebuild.
  8. Iterate on taste. Realistic proportions from reference images, then body smoothing, meadow depth, more grass, daylight. Every round: screenshot, feedback, adjustment — the loop a PM runs with any team.

Incidents & lessons

  • The frozen world. Hidden browser tabs pause animation, which silently froze automated tests. Fix: a deterministic step() seam so the simulation can be driven without the screen. Lesson: if a system only moves when someone is watching, it cannot be tested.
  • The stalled agent. A builder hit a timeout mid-milestone. Instead of restarting (and redoing finished work), it was resumed with a message anchored to its own last status report. Lesson: recovery beats restart when the work is resumable — and status reports are what make work resumable.
  • The pancake. Animation code hardcoded a measurement belonging to an older model of the dog; when the model changed, the tongue silently became enormous. Lesson: contracts between systems must live in one place. Also: always look at the thing.

The paper trail

Everything above is documented in real artifacts, the way a real product is: a design specification, an architecture plan, agent briefs with scope contracts, a persistent observation log of process improvements, and the telemetry in the corner of your screen. The dog is the demo; the discipline is the product.

Planned expeditions

A living roadmap, maintained the way roadmaps should be: honest about sequencing, sized by effort, and unashamed of the fact that everything on it is about a dog.

  • shipped Drivable Mini, vegetable market, fetch court, specimen dossiers, drag-to-orbit camera, daylight meadow, persistent telemetry.

Next

  • M Mini's day, live. A status feed her human updates from a phone — mood, crimes committed, vegetables consumed — rendered as field-manual telemetry.
  • M Global good-dog telemetry. One shared counter: every fetch and snack by every visitor, ever. The permanent record goes public.
  • S Photo constellations. Real photos of Mini scattered through the world as discoverable collectibles.

Later

  • L The guestbook meadow. Visitors leave a treat and a note; each becomes a permanent object in the world. The Miniverse accretes mass from love.
  • L Talk to Mini. An AI-powered conversation in which all answers eventually route back to vegetables.
  • L Hyper-real Mini. A rigged, film-quality dog model. Under evaluation: the current Mini has charm the realistic one must beat, not merely replace.
